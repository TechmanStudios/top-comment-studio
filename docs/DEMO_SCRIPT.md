# Hackathon Demo Script

Use this to shape the final presentation.

## Demo Title

Top Comment Studio

## Opening Pitch

Most AI video tools start with a blank prompt. Top Comment Studio starts with the audience: it turns the top safe comment into a creator-reviewed Runway render path for the next Short.

## Target User

Short-form creators who want their audience to help choose what gets made next without giving up creative control.

## Demo Flow

1. Open `https://top-comment-studio.onrender.com/`.
2. Show the selected audience comment as the only editable demo input.
3. Open Director Controls briefly to show they are locked for the competition workflow.
4. Click `Generate render` only when ready to spend Runway credits.
5. Let the render page auto-refresh while explaining that the Runway secret stays server-side on Render.
6. When the page reaches `Render ready`, open the MP4 with `Open video`.

## Happy Path

```text
Input: "Make the AI build a floating city powered by storms."
Expected output: A render page that submits the paid-proven v66 Gen/Veo workflow, auto-refreshes status, and exposes the final 1080x1920 MP4.
What to say: "The audience gave us the direction. The app keeps the creative controls stable, sends only the proven workflow inputs, and waits for a creator-approved Runway result."
```

## Backup Plan

If the live demo fails:

- Show screenshots or sample output.
- Explain the architecture.
- Show the working validation command.
- Walk through the intended flow.
- Use the previous paid-smoke MP4 if another live generation would risk time or credits.

## Judge-Friendly Talking Points

- What is novel?
- Why is this useful?
- What did we build during the hackathon?
- What is the technical architecture?
- What would we do next with more time?

## Final 20-Second Close

Top Comment Studio makes the audience part of the production loop. Each Short asks the community what should happen next, and the system turns the strongest safe signal into a creator-approved Runway render for the next episode.
