# Acceptance Criteria

Use this to keep hackathon work grounded.

## Repo Setup Is Done When

- [ ] The stack and package manager are documented.
- [ ] Dependencies can be installed from a fresh clone.
- [ ] `.env.example` exists with safe placeholder values.
- [ ] The app can run locally.
- [ ] At least one validation command exists and is documented.
- [ ] Known setup issues are documented.

## Pre-Build Setup Is Ready When

- [x] Project name and pitch are documented.
- [x] Public GitHub repository exists.
- [x] Safe `.env.example` exists.
- [x] Real secrets and browser session exports are ignored.
- [x] Runway docs, model catalog, workflows, and endpoint registry are documented.
- [x] Recommended first implementation stack is documented.
- [x] Next-phase build plan exists.
- [x] Runnable app scaffold exists.

## MVP Is Done When

- [ ] The primary user flow works end-to-end.
- [ ] The demo flow can be completed without editing code.
- [ ] Failure states are understandable to a user.
- [ ] The project has enough polish to explain in under two minutes.
- [ ] The next steps are obvious.

## AI Agent Work Is Acceptable When

- [ ] Changes are small and reviewable.
- [ ] The agent explains what changed.
- [ ] Validation commands were run or the reason they could not run is documented.
- [ ] No secrets were added.
- [ ] Any unresolved problems were added to `KNOWN_ISSUES.md`.
